# User Management

This guide explains how to manage users in Yet Another Wiki using the User Management settings panel.

:::info
User Management is only available with a Pro license.
:::

## User Management Interface 👥

### Accessing User Management

1. Navigate to Settings
2. Look for the "User Management" section
3. Requires Admin privileges

### Key Features

- Add/Edit/Delete users
- Manage user roles
- Control user registration
- Search functionality
- Pagination controls
- Bulk user selection

## User Registration Control 🔒

### Registration Settings

- Toggle "Prevent user registration" checkbox to:
  - Allow users to self-register
  - Restrict registration to admin-only
- Changes take effect immediately
- Affects the entire system

:::tip
Enable registration prevention if you want complete control over who can access your wiki.
:::

## Managing Users 👤

### Adding New Users

1. Click "Add User" button
2. Fill in required information:
   - Name
   - Email
   - Password
   - Role
   - Active status
3. Submit to create user

### Editing Users

Click the edit icon (✏️) to modify:
- User details
- Role assignments
- Account status
- Avatar
- Password (optional)

### Deleting Users

Two ways to delete:
1. Individual deletion:
   - Click delete icon (🗑️)
   - Confirm deletion
2. Bulk deletion:
   - Select multiple users
   - Use bulk actions

:::danger
User deletion is permanent and cannot be undone!
:::

## User Roles and Status 🎭

### Available Roles

- **Admin**: Full system access
- **Contributor**: Content management
- **User**: Basic access

### User Status

- **Active**: Can access the system
- **Inactive**: Access temporarily disabled

## Search and Navigation 🔍

### Search Functionality

- Real-time user search
- Search by:
  - Name
  - Email
  - Role

### Pagination

- Adjustable items per page:
  - 5 items
  - 10 items
  - 25 items
  - 50 items
- Navigate between pages
- Shows total pages and current position

## User List Features 📋

### Column Information

The user list displays:
1. User avatar and name
2. Email address
3. Role (with color coding)
4. Status (Active/Inactive)
5. Available actions

### Visual Indicators

- Role badges with distinct colors:
  - Admin: Purple
  - Contributor: Blue
  - User: Gray
- Status indicators:
  - Active: Green
  - Inactive: Red

## Bulk Actions ⚡

### Selection Options

- Select individual users
- Select all users on current page
- Clear selection

### Available Actions

- Bulk role assignment
- Bulk status changes
- Bulk deletion

## Best Practices 💡

1. **User Organization**:
   - Keep roles clearly defined
   - Regularly review access
   - Document role assignments

2. **Security**:
   - Regular access audits
   - Strong password policies
   - Monitor inactive accounts

3. **Maintenance**:
   - Clean up inactive users
   - Verify email addresses
   - Update role assignments

## Troubleshooting 🔧

### Common Issues

1. **User Can't Login**:
   - Check account status
   - Verify email address
   - Confirm role assignment

2. **Registration Issues**:
   - Check registration settings
   - Verify email configuration
   - Check license status

3. **Role Changes Not Applied**:
   - Refresh the page
   - Clear browser cache
   - Check permissions

:::tip
Keep your user list organized by regularly reviewing and updating user accounts and permissions.
:::

## Keyboard Shortcuts ⌨️

- `Shift + Click`: Select multiple users
- `Ctrl/Cmd + Click`: Individual selection
- `Esc`: Clear selection
